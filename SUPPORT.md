# Support

`jk` is early and maintainer-led.

Use GitHub issues for actionable bugs with a reproduction. Product-direction feedback is useful
when it references the current inspection loop, the product plan, or the roadmap. Broad feature
requests and implementation PRs are usually premature until the design settles.

Please do not file public issues for security reports. Use
[GitHub Security Advisories](https://github.com/joshka/jk/security/advisories/new) instead.
